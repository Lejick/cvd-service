Плавающие обновления позволяют обновить сервисы без простоев, шаг за шагом заменяя старые поды (контейнеры) на новые.
Новые поды будут запущены на узлах(виртуальных или реальных машинах), имеющих достаточно ресурсов.

Преимущества: 
-Осуществлять непрерывную доставку обновлений без простоев
-Переводить приложение из одного окружения в другое (через обновления образа контейнера)
-Откатываться к предыдущим версиям

Распространенным сочетанием являются docker образы, которые управляются системой kubernetes 


В общем процедуру можно описать следующим образом:
1. Поднимаем образы с версией v2
2. Переводим трафик с образов v1  на образы v2
3. При изменении протоколв взаимодействия между разными микросервисами переход от v1 к v2 должен осуществляться одновременно для этой группы сервисов
4. Постепенно переключаем трафик с сервисов v1 на v2. При обнаружении багов откатываем на v1
5. Гасим сервера с v1
